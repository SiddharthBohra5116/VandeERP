Password reset accepts empty/blank passwords
Critical
Security
adminController.js:resetPassword
▶
The resetPassword handler does absolutely no validation on the incoming req.body.password. An admin can submit a blank form and it will hash an empty string, effectively locking the user out — or worse, set a known weak password.

// CURRENT — no guards at all
exports.resetPassword = async (req, res) => {
  const u = await User.findById(req.params.id);
  u.password = req.body.password; // could be "", undefined, "a"
  u.resetRequested = false;
  await u.save();
};
// FIX — add validation before touching DB
exports.resetPassword = async (req, res) => {
  const { password } = req.body;
  if (!password || password.trim().length < 8) {
    return res.redirect(`/admin/dashboard?error=Password must be at least 8 characters`);
  }
  const u = await User.findById(req.params.id);
  if (!u) return res.redirect('/admin/dashboard?error=User not found');
  u.password = password.trim();
  u.resetRequested = false;
  await u.save();
};

Unescaped MongoDB $regex on user input — ReDoS risk
Critical
Security
adminController.js, counsellorController.js
▶
Every search field passes raw user input directly into { $regex: search, $options: 'i' }. A malicious user can craft a catastrophic backtracking regex (e.g. (a+)+b) to exhaust CPU — a classic ReDoS. Additionally, users can probe exact data by treating search as regex.

// CURRENT — dangerous
filter.name = { $regex: search, $options: 'i' }; // search is raw user input
// FIX — escape special regex chars
function escapeRegex(str) {
  return str.replace(/[.*+?^${}()|[\]\\]/g, '\\$&');
}
filter.name = { $regex: escapeRegex(search), $options: 'i' };

utils/sanitize.js
with an
escapeRegex()
helper
2
Import and apply in all 6 places that currently use raw $regex

  warn:  (msg, meta = {}) => console.warn(JSON.stringify({ level: 'warn',  msg, ...meta, ts: new Date() })),
  error: (msg, meta = {}) => console.error(JSON.stringify({ level: 'error', msg, ...meta, ts: new Date() })),
};
module.exports = logger;
